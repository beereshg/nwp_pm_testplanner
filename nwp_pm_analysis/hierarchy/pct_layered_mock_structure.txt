          availability) matches the SKU manifest. All tiles report identical

[Layer 1] PCT-CONTRACT-002
─────────────────────────────────────────────────────────────────────────────────
Title:    "PCT runtime toggle via SST_PP_CONTROL.feature_state[1] is honored"
Sentence: Writing SST_PP_CONTROL.feature_state[1] = 0 disables PCT (HP cores
          drop to base, LP cores unclip). Writing = 1 re-enables (HP cores
          return to HP TRL, LP cores re-clip). Transition completes within
          {manifest.pct_toggle_latency_us}.
§5 Bar:   Post-disable: all cores at uniform base ratio ±{manifest.ratio_tolerance};
          Post-re-enable: HP cores at {manifest.hp_trl} ±tolerance, LP at
          {manifest.lp_clip_ratio} ±tolerance.
Applicable: Parts where PCT-CONTRACT-001 == PASS on at least one partition; backends: silicon, simics.
Suspect:  PCode SST-TF flow (toggle path).
Gate:     PCT-CONTRACT-001.
Fan-out:  Per-backend × enable→disable, disable→enable sequences.

  TC instances:
    PCT-CONTRACT-002 / silicon / enable→disable / OS-isst
    PCT-CONTRACT-002 / silicon / disable→enable / OS-isst
    PCT-CONTRACT-002 / simics / enable→disable / scripted


[Layer 1] PCT-CONTRACT-003
─────────────────────────────────────────────────────────────────────────────────
Title:    "PCT negative: invalid CLOS/partition inputs are rejected or clamped"
Sentence: Out-of-range partition count (0, >4, non-power-of-2), writes to
          reserved CLOS fields, and direct TPMI writes bypassing BIOS either
          fault, are clamped to valid range, or produce no state change.
          Invalid inputs never corrupt live CLOS state.
§5 Bar:   Post-invalid-write: CLOS state == pre-write snapshot (bitwise);
          no silent corruption; error status register (if any) set.
Applicable: Parts where PCT-ENUM-001 == PASS; backends: silicon.
Suspect:  PCode input validation / TPMI access control.
Gate:     PCT-ENUM-001 (doesn't need CONTRACT-001 — tests the error path, not happy path).
Fan-out:  Per-invalid-input-class (out-of-range, reserved-field, bypass).

  TC instances:
    PCT-CONTRACT-003 / silicon / partition=0
    PCT-CONTRACT-003 / silicon / partition=5
    PCT-CONTRACT-003 / silicon / reserved-field-write
    PCT-CONTRACT-003 / silicon / direct-tpmi-bypass

[Layer 3] PCT-SCENARIO-002
─────────────────────────────────────────────────────────────────────────────────
Title:    "Ordered Throttle: LP cores throttle first under RAPL PL1"
Sentence: When RAPL PL1 is set below the power required to sustain HP TRL +
          LP clip simultaneously, PCode throttles LP cores first (reducing LP
          frequency toward Pn) before reducing HP cores below HP TRL.
§5 Bar:   Under sustained PL1 = {manifest.pl1_ordered_throttle_test_watts}:
          HP frequency ≥ {manifest.hp_trl} × 0.90 while LP frequency <
          {manifest.lp_clip_ratio} × 0.90 — for ≥ 10 seconds steady state.
Applicable: Parts where PCT-OBS-001 == PASS; backends: silicon.
Suspect:  PCode ordered-throttle priority logic.
Gate:     PCT-OBS-001.
Fan-out:  Per-PL1-level (mild throttle, severe throttle) × per-tile.

  TC instances:
    PCT-SCENARIO-002 / silicon / PL1=mild / CBB0
    PCT-SCENARIO-002 / silicon / PL1=severe / CBB0
    PCT-SCENARIO-002 / silicon / PL1=mild / CBB1
    PCT-SCENARIO-002 / silicon / PL1=severe / CBB1


[Layer 3] PCT-SCENARIO-003
─────────────────────────────────────────────────────────────────────────────────
Title:    "Thermal throttle respects HP/LP priority under PROCHOT"
Sentence: Under thermal throttle (PROCHOT or EMTTM), HP cores are protected
          longer than LP cores — LP frequency drops first. HP frequency
          reduction occurs only after LP is at minimum.
§5 Bar:   During thermal event: LP freq < {manifest.lp_clip_ratio} × 0.80
          while HP freq ≥ {manifest.hp_trl} × 0.85 — until thermal severity
          exceeds ordered-throttle range.
Applicable: Parts where PCT-OBS-001 == PASS; backends: silicon (requires real thermal injection).
Suspect:  PCode thermal × ordered-throttle interaction.
Gate:     PCT-OBS-001.
Fan-out:  Per-thermal-source (PROCHOT, EMTTM, DTS) × per-tile.

  TC instances:
    PCT-SCENARIO-003 / silicon / PROCHOT-inject / CBB0
    PCT-SCENARIO-003 / silicon / EMTTM-inject / CBB0
    PCT-SCENARIO-003 / silicon / PROCHOT-inject / CBB1


[Layer 3] PCT-SCENARIO-004
─────────────────────────────────────────────────────────────────────────────────
Title:    "DQ Rules: HP core disqualification promotes next-best core"
Sentence: When an HP core is disqualified (DQ'd via HWP or thermal event),
          PCode reassigns CLOS[0] to the next-highest-performance core from
          the favored-core ranking. The DQ'd core drops to LP clip.
§5 Bar:   Post-DQ: count(CLOS[0] cores) unchanged; new HP core frequency ≥
          {manifest.hp_trl} × 0.95; DQ'd core frequency ≤
          {manifest.lp_clip_ratio} × 1.05.
Applicable: Parts where PCT-FUSE-001 == PASS; backends: silicon.
Suspect:  PCode DQ reassignment logic / fuse-rank interpretation at runtime.
Gate:     PCT-FUSE-001 (fuse-derived ranking is the ground truth for promotion
          order — if fuses are wrong, DQ promotes the wrong core).
Fan-out:  Per-DQ-cause (thermal, HWP limit) × per-tile.

  TC instances:
    PCT-SCENARIO-004 / silicon / thermal-DQ / CBB0
    PCT-SCENARIO-004 / silicon / HWP-DQ / CBB0
    PCT-SCENARIO-004 / silicon / thermal-DQ / CBB1
