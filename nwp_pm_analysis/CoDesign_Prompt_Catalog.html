5. Key Rules (Always Enforced)

RuleDescription
40-line capStep 1a hierarchy snapshot must be ≤40 lines. If it overflows, run T2 to decompose the oversized TCD first.
Gist ruleDerive each TCD’s one-line WHAT from the scope description’s first paragraph, never the title.
Bar mandatoryEvery TCD entry in the snapshot MUST include its pass/fail bar. A snapshot without bars only gets structural answers.
NEW = bar requiredAny Co-Design new row without a spec-cited bar sketch is discarded — over-provisioning.
Never auto-pushCo-Design findings enter as Gap/TBD rows. Human confirms before any HSD write.
Spec ref → TCD §2Always copy the spec ref (clause ID) into TCD §2 — this is how FR linkage gets populated.
Co-Design = spec authorityCo-Design is authoritative for spec; local rules stay authoritative for structure.
Step 1b reuseSilicon routing context is write-once, reuse verbatim across all features. Only T6 overrides it with scope directive.
