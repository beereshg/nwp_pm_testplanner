TC22022422095 NEW [PSS] [Solar] DVFS-dvfs_all_random
env: virtual_platform cmd: —
▶ NWP Negative Stability Test — UFS ZBB Random DVFS Injection
⚠ NWP Scope Change (2026-06-10): Original Solar DVFS test is ZBB on NWP (Solar architecture absent). This TC is repurposed as a negative/stability test: randomly inject UFS_CONTROL TPMI writes while UFS is disabled (UFS_DISABLE fuse=1, mesh fixed at 2 GHz) and verify no crash, hang, or MCA. Solar tool replaced by PythonSV. Original description preserved below.
1. Validation Scope

Validates that NWP system stability is maintained when random Fabric DVFS UFS_CONTROL register writes are injected via TPMI while UFS is Zero Bug Baseline (UFS_DISABLE fuse=1). Primecode (SPMFWREQ-928) must hold the fabric at fused P1 (2 GHz) regardless of written TPMI content.

Negative test objectives:
  • No MCA / IERR / system hang across 100 random UFS_CONTROL writes
  • UFS_HEADER.AUTONOMOUS_UFS_DISABLED=1 persists (SPMFWREQ-927 regression)
  • UFS_STATUS.CURRENT_RATIO=20 (2.0 GHz) on both CBBs throughout (SPMFWREQ-928 regression)
  • PLR bit[22] (RACL) stays 0 throughout (ZBB guard)
PASS: All 100 iterations complete; 0 failures; system alive; no MCA.
FAIL: Any ratio deviation, RACL assertion, MCA, IERR, or system hang.
2. Preconditions
  • Platform booted to XOS / SVOS with PythonSV namednodes access
  • NWP silicon: 2 CBBs + 1 NIO die; UFS_DISABLE fuse=1 (customer default, HSD 14024876702)
  • BIOS knobs: all DVFS / UFS knobs at default; no manual UFS enable override
  • No sustained workload driving power above PL2 (avoids confounding PLR bits)
  • PythonSV sv namespace loaded; SVDEFS kernel module loaded
  • Solar tool not required — test uses PythonSV TPMI injection only
3. Test Steps
Step 1 — Gate check: verify UFS ZBB state before injection:
import random, time # NWP CBB TPMI paths cbb0_hdr = sv.socket0.cbb0.base.tpmi.ufs_header cbb1_hdr = sv.socket0.cbb1.base.tpmi.ufs_header cbb0_sts = sv.socket0.cbb0.base.tpmi.ufs_status cbb1_sts = sv.socket0.cbb1.base.tpmi.ufs_status cbb0_ctl = sv.socket0.cbb0.base.tpmi.ufs_control cbb1_ctl = sv.socket0.cbb1.base.tpmi.ufs_control plr = sv.socket0.nio.punit.ptpcfsms.ptpcfsms.perf_limit_reasons # Gate: AUTONOMOUS_UFS_DISABLED must be 1 on both CBBs (SPMFWREQ-927) for idx, hdr_reg in enumerate([cbb0_hdr, cbb1_hdr]): val = hdr_reg.read() dis = (val >> 8) & 0x1 # bit[8] = AUTONOMOUS_UFS_DISABLED assert dis == 1, 'CBB' + str(idx) + ' AUTONOMOUS_UFS_DISABLED=' + str(dis) + ' (expected 1)' print('CBB' + str(idx) + ' AUTONOMOUS_UFS_DISABLED =', dis, ' [expected 1]') # Gate: CURRENT_RATIO must be 20 = 2.0 GHz (unit 100 MHz) for idx, sts_reg in enumerate([cbb0_sts, cbb1_sts]): ratio = sts_reg.read() & 0x7F # bits[6:0] assert ratio == 20, 'CBB' + str(idx) + ' pre-test ratio=' + str(ratio) + ' (expected 20)' print('CBB' + str(idx) + ' CURRENT_RATIO =', ratio, ' [expected 20 = 2.0 GHz]')
PASS: Both CBBs AUTONOMOUS_UFS_DISABLED=1 and CURRENT_RATIO=20.
FAIL: Either assertion fires — stop; log UFS_HEADER raw value and fuse state.

Step 2 — Inject 100 random UFS_CONTROL writes; verify stability per iteration:
P0_RATIO = 20 # 2.0 GHz / 100 MHz unit Pn_RATIO = 8 # 800 MHz minimum ITERATIONS = 100 failures = [] for i in range(ITERATIONS): rnd_min = random.randint(Pn_RATIO, P0_RATIO) rnd_max = random.randint(rnd_min, P0_RATIO) # UFS_CONTROL: MAX_RATIO[14:8], MIN_RATIO[21:15] ctrl_val = ((rnd_max & 0x7F) << 8) | ((rnd_min & 0x7F) << 15) cbb0_ctl.write(ctrl_val) cbb1_ctl.write(ctrl_val) time.sleep(0.02) # 20 ms = one Primecode slow-loop # Check 1: CURRENT_RATIO must stay at P0 (ZBB enforcement) for idx, sts_reg in enumerate([cbb0_sts, cbb1_sts]): ratio = sts_reg.read() & 0x7F if ratio != P0_RATIO: failures.append('iter=' + str(i) + ' CBB' + str(idx) + ': ratio=' + str(ratio)) # Check 2: RACL PLR bit[22] must stay 0 (ZBB guard) if (plr.read() >> 22) & 0x1: failures.append('iter=' + str(i) + ': RACL PLR bit[22] asserted') if (i + 1) % 10 == 0: tag = 'no failures' if not failures else str(len(failures)) + ' failures' print('Iter', i+1, '/100 -', tag) print('Completed', ITERATIONS, 'iterations. Total failures:', len(failures)) for msg in failures: print(' FAIL:', msg)
PASS: failures=[] after 100 iterations; system responding.
FAIL: Any CURRENT_RATIO deviation or RACL assertion — ZBB enforcement broken in CBB PCode.

Step 3 — Restore UFS_CONTROL to default (0 = driven by fused P0/Pn):
cbb0_ctl.write(0) # 0 = MAX/MIN from fused P0/Pn cbb1_ctl.write(0) time.sleep(0.1) for idx, sts_reg in enumerate([cbb0_sts, cbb1_sts]): ratio = sts_reg.read() & 0x7F print('Post-restore CBB' + str(idx) + ' CURRENT_RATIO =', ratio, ' (expected 20)')
PASS: Both CBBs return CURRENT_RATIO=20 after restore.
FAIL: Ratio does not recover — Primecode/PCode UFS state machine may be stuck.

Step 4 — System stability: MCA / IERR scan:
import subprocess result = subprocess.run(['dmesg', '-T', '--level=crit,err'], capture_output=True, text=True) mca_lines = [l for l in result.stdout.splitlines() if any(k in l for k in ['MCA', 'IERR', 'Machine Check'])] print('MCA/IERR entries found:', len(mca_lines)) for line in mca_lines: print(line)
PASS: 0 MCA/IERR entries post-test.
FAIL: Any MCA or IERR — capture full dmesg and Primecode trace log before reset.

5. Post-Process
  • failures list with CURRENT_RATIO ≠ 20: CBB PCode UFS ZBB enforcement broken (SPMFWREQ-928 violation) → file sighting against CBB PCode
  • PLR bit[22]=1: RACL activated on ZBB system → file sighting against Primecode TPMI init (SPMFWREQ-927)
  • System hang: capture last HPM 0x1b message via PMSB trackers before reset
  • Confirm UFS_HEADER.AUTONOMOUS_UFS_DISABLED=1 persists across all 100 writes
