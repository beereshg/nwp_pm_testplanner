# Core C-States > C1

> **Status**: Enriched — HW/FW/OS touchpoints + KPI & Timing added (2026-05-28)
> **Parent**: [Core C-States](core_c_states_main.md)

## Baseline (DMR)

Core C1 is the **first real idle state** — the core clock is gated, shutting down the core's operation while maintaining full voltage and PLL frequency. C1 is the **fastest** entry/exit C-state because no state save, cache flush, or power-gate occurs.

## HW Touchpoints

| IP Block | Die | Role | Key Signals / Wires | HAS Reference |
|----------|-----|------|---------------------|---------------|
| ACP / PMA | CBB | Core-local C-state FSM: clock-gates core when thread reaches TC1/CC1; exit triggered by break event | core_active deassert on clock-gate; cfcclk partial gate | PNC PM HAS §8.9 |
| PICLET / FIL | CBB | Not active in C1 — interrupt filtering only in C6; TC1 exit is direct break-event path | break_event passes through; piclet_bypass in C1 | PNC PM HAS §8.9.2 |
| FIVR | CBB | Holds core voltage at full operating point throughout CC1; drops to LFM only on C1E promotion | VCCcore maintained at operating level; FIVR control from PMA | DMR CBB PM HAS |
| CTC / Crystal Clock | CBB | Continues running in CC1; drives residency counter MSR 0x778 (crystal cycles) | CTC_active = 1 throughout C1; residency counter increment | PNC PM HAS §8.3 |

## FW Touchpoints

| Agent | Location | Role | Key Functions / Handlers | Source |
|-------|----------|------|--------------------------|--------|
| uCode (PantherCove) | CBB core | Thread C-state sequencer: MWAIT/HLT dispatch, TC1 entry flow (10 steps), APIC counter freeze, event inhibits | TC1 entry: CPL check → Monitor check → freeze APIC → inhibits → sleep_nuke; exit on break event | PNC PM HAS §8.9 |
| Acode (Tensilica ACP) | CBB core perimeter | C1 demotion algorithm; C1E WP calculation for Fast C1E; monitors ~CoreActive for C1 incidency | C1E promotion tracking (8-entry sliding window); C1_DEMOTION via UCODE_CFG.WISH_ALLOW | PNC PM HAS §8.9; §8.10 |
| PCode (CBB) | CBB | Slow C1E: detects C1E condition after 64–250 µs; propagates RESOLUTION_CONTROL.WISH_ALLOW to PMA | WISH_ALLOW population; MSR 0xE2 reflection; C1E detection slow-path | DMR CBB PM HAS |
| BIOS / UEFI | Platform | Programs C-state enable knobs at boot; configures ACPI _CST with C1 descriptor | MonitorMWait enable; C1 ACPI descriptor population; MSR 0xE2 init | BIOS PM HAS |

## OS Interfaces

| Interface | ID / Address | Access | Description | Spec Reference |
|-----------|-------------|--------|-------------|----------------|
| MWAIT / HLT | CPU instruction | W | Requests C1 state; EAX[7:4]=0x0 (C1), sub-state 0=keep VR / 1=C1E (lose VR) | IA-32 SDM; PNC PM HAS §8.1 |
| MSR POWER_CTL | 0x1FC | RW | bit[0] C1E_ENABLE — enables OS C1→C1E promotion via MWAIT sub-state 1 | PNC PM HAS §8.10 |
| MSR CLOCK_CST_CONFIG_CONTROL | 0xE2 | RW | bit[26] C1_DEMOTION_ENABLE, bit[28] ENABLE_C1_UNDEMOTION | PNC PM HAS §8.6.5 |
| MSR CC1 Residency | 0x778 | RO | Crystal cycles spent in CC1 (C1 and C1E combined — no distinction) | PNC PM HAS §8.3 |
| MSR CC1 Residency (alt) | 0x660 | RO | CC1 residency in x1 clocks (alternate implementation-specific counter) | PNC PM HAS §8.3.5 |
| CPUID.05H | Leaf 5, EDX | RO | Enumerates supported C-state sub-states; C1 sub-state 0 (keep VR) and 1 (C1E) | IA-32 SDM |
| ACPI _CST | ACPI namespace | RO | OS C-state descriptor for C1; BIOS-populated per logical processor | ACPI Spec 6.x |

## NWP Delta

**C1 core idle is fully supported on NWP** — same PantherCove (PNC) core as DMR; no change.

- C1 is a standard halt/clock-gate state; snoops processed, caches not flushed
- Same FW agents as DMR: uCode, Acode (ACP), PCode manage C1 — no architectural difference
- **PkgC6 removed** on NWP; C1 behavior is unaffected

### Validation Impact
- Same C1 test cases apply as DMR
