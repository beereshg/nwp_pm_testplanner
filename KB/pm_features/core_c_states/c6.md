# Core C-States > C6

## Baseline (DMR)

Core C6 is the **power-gated** idle state where the core's architectural state is saved to **C6SRAM** (a dedicated region in MLC), clocks are shut off, isolation is enabled, and PowerGood is deasserted. C6 exits via a **staged flow**: CC6→CC1 (reset handler restores state) then CC1→CC0 (break event wakes thread).

## NWP Delta

**C6 deep core idle is supported on NWP** — same PantherCove (PNC) core as DMR.

### Architecture Changes
- NWP uses the same PantherCove (PNC) P-core as DMR — ACP/Acode manages C6 enter/exit as on DMR
- Core-level CC6 behavior unchanged: PICLET/FIL, C6SRAM save/restore, budget negotiation flow all same
- **PkgC6 is removed** on NWP — MC6 becomes the deepest idle state (no deeper package power-gate)

## Legacy (Human-Curated Reference)

### Architecture Summary

Core C6 is the **power-gated** idle state where the core's architectural state is saved to **C6SRAM** (a dedicated region in MLC), clocks are shut off, isolation is enabled, and PowerGood is deasserted. C6 exits via a **staged flow**: CC6→CC1 (reset handler restores state) then CC1→CC0 (break event wakes thread).

#### C6 Flavors (PNC Enumeration) *(confirmed: [PNC PM HAS §8.1.2](c:\github\NWP\PNC%20PM%20HAS%200.5_text.txt))*

| Flavor | MWAIT Code | Cache Flush | PkgC | Autonomous | Budget Released | Use Case |
|--------|-----------|-------------|------|------------|----------------|----------|
| **C6** | 0x22 | No (MLC valid) | PC0 | No | Yes (full) | Standard C6 — minimum latency, no MLC flush |
| **C6A** | 0x24 | No | PC0 | **Yes** | **No** | Autonomous exit — PMA wakes core without PUnit; lowest exit latency |
| **C6S** | 0x23 | **Yes** (MLC+L2+LLC) | PC0 | No | Yes (full) | C6 with cache flush — enables MC6 |
| **C6S-P** | 0x20 | **Yes** | **PC6** | No | Yes (full) | C6 with flush + Package C-state allowed; deepest non-autonomous |
| **C6SA** | 0x25 | **Yes** | PC0 | **Yes** | **No** | Autonomous with cache flush |

**C6A (Autonomous)** is the key PNC innovation: the PMA retains the electrical budget when core enters C6, so C6 exit bypasses PUnit budget negotiation entirely. This dramatically reduces exit latency for latency-sensitive workloads that disable PkgC.

#### State Save/Restore Areas

| Content | Save Location | When Saved | When Restored |
|---------|--------------|-----------|---------------|
| GP registers, FP/SSE state, patch state | C6SRAM (MLC region) | TC6 entry (thread-specific) | CC6 exit (primary thread reset handler) |
| Thread-specific control registers, SMM state | C6SRAM | TC6 entry | CC6 exit |
| Core control registers (CRegs) | C6SRAM | CC6 coordination | CC6 exit |
| APIC state | C6SRAM | CC6 coordination (after point-of-no-return) | CC6 exit |
| Module shared MLC content | C6SRAM → DRAM (C6DRAM) | MC6 entry (if C6DRAM enabled) | MC6 exit (first core) |

C6SRAM base address per core is set by PM at reset via `ML3_CR_PIC_BOOT_MESSAGE`.

#### PICLET (Interrupt Filtering) *(confirmed: [PNC PM HAS §8.12.2](c:\github\NWP\PNC%20PM%20HAS%200.5_text.txt))*

During CC6, interrupts are filtered by PICLET (in FIL). When a break event arrives:
1. PICLET recognizes it and sends **wake request** to PMA
2. PMA issues **Reset** or **Wake** event to core (based on current Core_Status)
3. uCode handles reset event → disables PICLET → triggers **PICLET replay**
4. FIL replays queued interrupts back to APIC, disables redirection, unblocks U2C path
5. Real break event reaches APIC → thread wakes

#### Auto C-States (ACP/PMA-managed) *(confirmed: [PNC PM HAS §8.5](c:\github\NWP\PNC%20PM%20HAS%200.5_text.txt))*

In PNC (LNC/DMR/NWP), C-state management moved from PUnit to the **Core Perimeter** (ACP):
- ACP/PMA owns C-state entry/exit/demotion flow
- Autonomous C6 exit improves interrupt service time and time-to-full-performance
- Resource negotiation uses 3× Q-channels + Budget negotiation (Type A/A'/B electrical requests)
- Punit is no longer the bottleneck for C-state entry/exit latency (critical for scalability as core count increases)

#### BIOS Knob Flow

```
1. BIOS knobs configure:
   ├── C6Enable → enables/disables C6 entry via MSR 0xE2 (CLOCK_CST_CONFIG_CONTROL)
   ├── C1AutoDemotion → enables C6-to-C1 demotion
   ├── MonitorMWait → enables MONITOR/MWAIT pair
   └── C1e → enables C1E autopromotion

2. PUnit propagates MSR 0xE2 to PMA_CR_RESOLUTION_CONTROL.WISH_ALLOW

3. At runtime:
   ├── uCode reads CORE_PMA_CR_IO_UCODE_CFG.WISH_ALLOW before entering C6
   ├── If WISH_ALLOW < C6 (demotion): uCode enters C1 instead
   └── If WISH_ALLOW >= C6: proceed with C6 entry
```
