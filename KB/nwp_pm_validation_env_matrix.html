Environment Selection Rules (derived from test content)

🖥 Simics (Virtual Platform) — strongest when TCDs are about:
  • Register / BIOS knob / fuse reads and writes
  • Discovery and interface programming
  • Telemetry, reporting, interrupt plumbing
  • SW-visible state at register/MSR level
  • Timing correlation at SW/reporting level
Best candidates: PM Interfaces, TPMI, PECI, PMT, SVID, PLR, PEM, Thermal Interrupts, RAPL register/SW interface
🧠 SLE (Core IP) — strongest when TCDs are about:
  • Core entry/exit actions (P-state, C-state)
  • HWP / hardware autonomous frequency selection
  • Core counters / residency at core level
  • Core-local throttling / ICCP / VF policy
  • Fast C1E, demotion, module C-state behavior
Best candidates: Core P-states, HWP, Turbo, ICCP, SAPM-DLL, EET, Core C-states, Fast C1E, Demotion, Module C-states
📡 HSLE (System/Package) — strongest when TCDs are about:
  • HPM messaging and package flows
  • IMH / CBB / NIO / fabric interactions
  • DVFS in uncore / fabric / memory
  • Thermal / package / cross-product integration
  • OOB / endpoint sweep / PM integration suites
Best candidates: PKGC, Fabric DVFS, CBB CCF PM, SoC Thermal, ITD, Memory Thermal, AIPM, SST/PCT, SIMPL, PM Cross Product
🔬 Silicon — mandatory when TCDs imply:
  • Real power / current / voltage / electrical behavior
  • Real thermal behavior (temperature ramps, gradients)
  • Residency and performance realism
  • Tuning / calibration / fuse closure
  • Power-limiting real-time dynamics
Best candidates: Socket RAPL, PMAX, ITD closure, SoC Thermal closure, Memory Thermal closure, RACL, Core FIVR
