What is ITD?

ITD (Inverse Temperature Dependence) is a temperature-dependent voltage compensation mechanism — not a frequency-throttle feature. It corrects per-domain FIVR/MBVR voltage targets to track the physics of how transistor behavior changes with temperature at low voltages.

Physics basis: At low voltages, carrier mobility increases as temperature drops (beneficial), but threshold voltage also increases (harmful). Below the ITD crossover point, the net effect is that cold silicon needs more voltage at the same frequency. Without dynamic compensation, a fixed worst-case static guardband wastes significant power margin at nominal temperatures.

Algorithm (per-domain, per evaluation tick)

V_comp = slope × (ITD_CUTOFF_V − V_uncompensated) × (ITD_CUTOFF_TJ − T_DTS)
Dual-slope: slope1 if V < ITD_CUTOFF_V_X, slope2 if V ≥ ITD_CUTOFF_V_X
True TD (TTD): negative compensation for hot silicon (enabled by TRUE_TD_ENABLE fuse)
Result written to: RESCTRL_CR_VOLTAGE_OFFSET.OFFSET per RC channel

Complexity Rating: Medium-High

  • Algorithmic: 2-slope piecewise + True TD + per-domain fuse interpretation
  • Domain/rail mapping: 3 firmware agents, 12+ domains, NWP adds VCCCAB + VCCC2CDIG
  • Interaction: PkgC, D2D handshake, reset/unwind, DCM concurrency
  • Validation: Not smoke-test only — requires algorithm correctness + electrical + interaction testing
  • Estimate: ~3 weeks class feature
