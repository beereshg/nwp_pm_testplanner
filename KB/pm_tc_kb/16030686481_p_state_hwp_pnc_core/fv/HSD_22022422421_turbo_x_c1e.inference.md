## Section A: NWP Disposition & Justification

**Disposition: Runnable_On_N-1**

This test verifies **Turbo × C1E** cross-product — interaction between Turbo Mode and C1E (Enhanced Halt — frequency and voltage reduction on halt). C1E is a lightweight idle state that coexists with Turbo.

C1E is a standard core idle state, functional on NWP. The cross-product tests that Turbo is correctly granted or adjusted when some cores are in C1E.

**Key Justification:**
- C1E is functional on NWP (not ZBB)
- `DMR_PO` + `plc.ti_gate.b0` + `plc.feature.p1` + `pm.xproducts.pm` + `PMSS_NWP_READINESS_CHECK` tags
- `pm.xproducts.pm` explicitly marks this as a PM cross-product test

## Section B: NWP-Specific Test Procedure

### C1E in NWP Context

C1E: core executes `HLT` instruction → PCode reduces frequency/voltage. Core remains ready but at lower power.

On NWP (non-SMT): C1E per physical core. No thread-level C1E aggregation needed.

### Adapted Test Steps

| Step | Action | NWP Details |
|------|--------|-------------|
| 1 | Enable C1E: `PowerManagement.C1E = 1` BIOS knob | Standard BIOS config |
| 2 | Enable Turbo | `TurboMode = 0x1` |
| 3 | Run turbo PMx | `python runPmx.py -x nwp.xml -p turbo -tM 60` |
| 4 | Stress subset of cores; let others idle to C1E | Mixed active/idle workload |
| 5 | Verify active cores receive Turbo | `IA32_PERF_STATUS` shows turbo ratio |
| 6 | Verify C1E cores properly halt | Power reduction on halted cores |
| 7 | Verify no instability on C1E entry/exit | No hang or crash on wake |

### NWP Pass Criteria
- Turbo granted to active cores while idle cores in C1E
- C1E frequency reduction occurs on halted cores
- Active cores maintain turbo during C1E partial idle condition
- No power management errors during transitions
