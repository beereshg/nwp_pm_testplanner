## Section A: NWP Disposition & Justification

**Disposition: Runnable_On_N-1**

This test verifies **Turbo × C6S** cross-product — interaction between Turbo Mode and C6S (C6 with LLC Save — deeper than C6A). In C6S, core powers off and the LLC associated with that core is flushed/saved, enabling deeper power reduction.

C6S is a core-level idle state functional on NWP. Not to be confused with Package C6 or Ring C6/MC6 (both ZBB on NWP).

**Key Justification:**
- Core C6S functional on NWP (not ZBB)
- `DMR_PO` + `plc.ti_gate.b0` + `plc.feature.p1` + `pm.xproducts.pm` tags
- C6S provides deeper idle than C6A (LLC flush) → more Turbo headroom

## Section B: NWP-Specific Test Procedure

### C6A vs C6S on NWP

| C-State | LLC Behavior | Power Savings |
|---------|-------------|---------------|
| C6A | L1 flush; LLC may remain | Less |
| **C6S** | L1 + LLC flush/save | **More** |
| Pkg C6 | Full package off | ZBB on NWP |

### Adapted Test Steps

| Step | Action | NWP Details |
|------|--------|-------------|
| 1 | Enable C6S BIOS knob; enable Turbo | `C6Enable = 1`, `TurboMode = 0x1` |
| 2 | Run turbo PMx | `python runPmx.py -x nwp.xml -p turbo -tM 60` |
| 3 | Mixed workload: subset active, rest idle to C6S | `stress-ng` on subset |
| 4 | Verify active cores get Turbo | `IA32_PERF_STATUS` |
| 5 | Verify idle cores reach C6S (LLC flush) | C-state status + LLC state |
| 6 | Verify C6S exit on interrupt (no hang) | Wake latency acceptable |

### NWP Pass Criteria
- Turbo granted to active cores while others in C6S
- C6S (with LLC flush) functional on NWP
- C6S exit latency within spec; no hang on wake
- Turbo × C6S does not cause PM instability
