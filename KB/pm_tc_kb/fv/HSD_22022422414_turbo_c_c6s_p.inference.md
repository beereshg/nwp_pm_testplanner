## Section A: NWP Disposition & Justification

**Disposition: Runnable_On_N-1**

This test verifies **Turbo × C6S-P** cross-product — interaction between Turbo Mode and the C6S-P (C6 with LLC flush, Package-oriented) idle state. Turbo can be granted/denied based on idle state transitions.

**C6S-P on NWP**: Core C6 states (C6A, C6S, C6S-P) are distinct from Package C6 (`PkgC6`) and Ring C6/MC6. Only `PkgC6` and `Ring C6/MC6` are ZBB on NWP. Core C6 states are functional on NWP.

**Key Justification:**
- Core C6S-P (LLC flush) functional on NWP (not ZBB)
- `DMR_PO` + `plc.feature.p1` + `PMSS_NWP_READINESS_CHECK` tags
- Turbo + C-state cross-product verifies power management interaction
