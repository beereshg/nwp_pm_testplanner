## Section A: NWP Disposition & Justification

**Disposition: Runnable_On_N-1**

This test verifies **Turbo × C6A** cross-product — interaction between Turbo Mode and C6A (Autonomous C6 — core powered off autonomously, L1 cache flushed). C6A is deeper than C1E and allows greater power savings for idle cores, potentially increasing Turbo headroom for active cores.

C6A is a core-level idle state. Only Package C6 and Ring C6/MC6 are ZBB on NWP; core-level C6 states (C6A, C6S, C6S-P) are functional.

**Key Justification:**
- Core C6A functional on NWP (not ZBB)
- `DMR_PO` + `plc.ti_gate.b0` + `plc.feature.p1` + `pm.xproducts.pm` tags
- C6A saves more power than C1E → more Turbo headroom for active cores

## Section B: NWP-Specific Test Procedure

### C6A in NWP Context

C6A: core autonomously powers off (including L1) without OS involvement. Core wakes on interrupt. PCode tracks C6A exit latency for wakeup management.

### Adapted Test Steps

| Step | Action | NWP Details |
|------|--------|-------------|
| 1 | Enable C6A via BIOS: `C6Enable = 1` | Standard BIOS |
| 2 | Enable Turbo | `TurboMode = 0x1` |
| 3 | Run turbo PMx | `python runPmx.py -x nwp.xml -p turbo -tM 60` |
| 4 | Run mixed workload: some cores fully active, others idle to C6A | e.g., `stress-ng` on subset |
| 5 | Verify active cores receive Turbo boost | `IA32_PERF_STATUS` shows turbo ratio |
| 6 | Verify idle cores reach C6A | Check `cbb.compute0.cpu.module0.core0.pkg_cstate_status` |
| 7 | Verify C6A exit on wakeup event (no hang) | Interrupt delivery to C6A core |

### NWP Pass Criteria
- Active cores get Turbo; idle cores in C6A reduces platform power
- C6A entry/exit functional on all NWP cores
- No wake latency issues or hang on C6A exit
- Turbo budget preserved or expanded when cores in C6A
