## Section A: NWP Disposition & Justification

**Disposition: Runnable**

**DCM FIT (Dual Core Module Frequency Isolation Tolerance)** is an intra-die feature that allows frequency isolation between the two PantherCove (PNC) cores within a single Dual-Core Module (DCM) that share the same PLL and FIVR. Without FIT, if one core runs a high-cdyn workload (e.g. AVX-512), both cores in the DCM are forced to the same lower AVX-512 TRL floor — the "noisy neighbor" effect. DCM FIT lets software (BIOS/hypervisor) program a per-DCM frequency variability tolerance via TPMI, expressed in units of 100 MHz relative to the SSE Turbo Frequency (SSE P0x). PCode enforces the tolerance so the lighter-workload core can run within FIT units of its SSE ceiling, independent of the heavier core's ICCP license level.

> ⚠️ **KB Correction (2026-07-17)**: The feature file `pstate_dcm_fit.md` previously described this as "Dual Chip Module Frequency Integration Technology" (die-to-die coordination). That is **incorrect** for this TC. This TC targets the **Dual Core Module (intra-die, per-CBB)** FIT — two PNC cores within one physical module sharing PLL/FIVR. NWP has 24 DCMs × 2 PNC cores per CBB (2 CBBs = 96 cores total), so this feature is **fully applicable on NWP**.

**NWP DCM Topology Confirmed:**
- NWP has 2 CBBs (die 0–1), each with 48 PNC cores
- 24 DCMs per CBB; each DCM contains 2 PNC cores sharing PLL and FIVR
- FIT mechanism (TPMI per-DCM tolerance register, pCode enforcement) carries over directly from DMR
- Only required adaptation: `dmr.xml` → `nwp.xml` in automation command

### DCM FIT Mechanism

| Parameter | DMR | NWP |
|-----------|-----|-----|

| FIT enforcement agent | pCode PEGA | Same |

### Key Registers

| Register | Description | NWP Path |
|----------|-------------|----------|
| `IA32_PERF_STATUS (0x198)` | Current core operating ratio | `sv.socket0.cbb[0-1].compute[x].module[y].core[0-1].ia32_perf_status` |
| `IA32_CORE_PERF_LIMIT_REASONS (0x64F)` | Per-core PLR: AVX throttle bit, FIT-related limiting reasons | Same MSR per core |
| TPMI FIT tolerance register | Per-DCM FIT tolerance in 100 MHz units (0 = disabled) | NWP TPMI MMIO |

### Test Steps

| # | Action | Expected Result | NWP Adaptation |
|---|--------|-----------------|----------------|
| 1 | Boot with Turbo enabled; identify a target DCM (2 PNC cores in same module). Record SSE Turbo Frequency (SSE P0x) via `IA32_PERF_STATUS` under SSE-only load | System boots; SSE P0x readable | No change |
| 2 | **Baseline (FIT disabled)**: run AVX-512 on Core 0 + SSE on Core 1 of same DCM simultaneously | Both cores forced to AVX-512 TRL (noisy neighbor) | No change |
| 3 | Program TPMI FIT tolerance = 1 (100 MHz below SSE P0x) for target DCM; verify register readback | TPMI reads back as 1; pCode acknowledges | NWP TPMI path |
| 4 | **FIT active**: repeat mixed workload; read `IA32_PERF_STATUS` per core | Core 0: AVX-512 TRL; Core 1: ≥ SSE P0x − 100 MHz | No change |
| 5 | Check `IA32_CORE_PERF_LIMIT_REASONS (0x64F)` per core | Core 0: AVX throttle set; Core 1: no AVX throttle bit | No change |
| 6 | Optionally sweep FIT tolerance (2, 3, N×100 MHz) | Core 1 ceiling = SSE P0x − FIT×100 MHz for each value | No change |
| 7 | Run end-to-end automation | PASS; all FIT isolation and PLR checks pass | `nwp.xml` |

### NWP PythonSV — Per-Core Ratio Readout

```python
# NWP DCM FIT: read per-core ratios across both CBBs
import namednodes as nn

sv = nn.sv
for cbb_idx in range(2):  # NWP: 2 CBBs
    cbb = getattr(sv.socket0, f'cbb{cbb_idx}')
    for compute_idx in range(4):    # 4 compute clusters per CBB
        for module_idx in range(6): # 6 modules per compute (24 DCMs / 4 computes)
            try:
                mod = cbb.getbypath(f'compute[{compute_idx}].module[{module_idx}]')
                c0 = (mod.core[0].ia32_perf_status.read() >> 8) & 0xFF
                c1 = (mod.core[1].ia32_perf_status.read() >> 8) & 0xFF
                if abs(c0 - c1) > 1:
                    print(f'CBB{cbb_idx} CP{compute_idx} M{module_idx}: core0={c0} core1={c1} delta={abs(c0-c1)}')
            except Exception:
                pass
```

### Pass Criteria

- With FIT active: Core 1 (light workload) frequency ≥ SSE P0x − (FIT_tolerance × 100 MHz); Core 1 not coupled to Core 0's AVX-512 TRL
- Without FIT: Core 1 forced to same AVX-512 TRL as Core 0 (baseline confirmed)
- TPMI FIT register persists across workload transitions
- `IA32_CORE_PERF_LIMIT_REASONS`: Core 0 has AVX throttle bit set; Core 1 does not
- No MCA, soft hang, or BIOS error during mixed-workload DCM stress
- Automation: `runPmx.py -x nwp.xml -p fit_check -tM 60` returns PASS
