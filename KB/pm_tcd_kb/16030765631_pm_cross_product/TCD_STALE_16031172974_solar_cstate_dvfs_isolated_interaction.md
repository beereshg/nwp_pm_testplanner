## Section 1: Architecture / Micro-architecture and Functionality

Validates that C-state entry/exit transitions and DVFS workpoint changes coexist correctly when no P-state stimulus is present. This isolates the C-state/DVFS interaction from P-state arbitration complexity, testing the specific spec requirement that C-state changes have no impact on DVFS request completion.

> **Architecture overview:** See [TPF 22022562325 — PM Integration Testing](https://hsdes.intel.com/appstore/article-one/#/22022562325) Section 2 Design Details for cross-product interaction architecture.

### NWP-Specific Deltas

- NWP has 2 CBBs (96 cores) vs DMR up to 4 CBBs
- PkgC6 is ZBB on NWP — C-state depth limited to CC6
- Fabric DVFS on NWP follows DMR architecture (CBB-side)
- No P-state changes injected — isolates the C-state/DVFS path

## Section 3: Reset, Power, and Clocking

- Scenario begins after full boot; P-state is held constant (no HWP/legacy changes)
- C-state entry gates core clock domain; DVFS operates on fabric/uncore domain
- C-state exit triggers clock ungating; DVFS request must not be lost or corrupted
- This scenario specifically validates the clock-domain boundary interaction

---

## Section 4: Programming Model

DVFS requests are generated by PCode based on workload demand and power budget. C-state entry/exit is driven by OS idle injection. When no P-state changes are present, the only frequency variable is DVFS — this isolates testing of the spec requirement: "C-State Change should not have any impact on [DVFS] request."

The test confirms DVFS completion is not blocked, delayed beyond spec, or corrupted by concurrent C-state transitions in the core clock domain.

## Section 5: Operational Behavior

### Pass/Fail Bar

| Criterion | Measurable Threshold | Spec Basis |
|-----------|---------------------|-----------|
| No MCA | Zero MCAs logged during and after stimulus | Architecture invariant |
| No hang | Test completes within timeout | Architecture invariant |
| DVFS completion | Every DVFS request completes and reaches commanded target frequency | DMR Fabric DVFS HAS: "C-State Change should not have impact on request" |
| Post-exit frequency | After C-state exit, effective frequency matches last DVFS target | CCP PM HAS: clock restoration on C-state exit |

## Section 6: Corner Cases & Error Handling

| Corner Case | Description | Current Coverage | Action Required |
|-------------|-------------|-----------------|----------------|
| DVFS transition during C6 entry | Clock gating overlaps with active DVFS | Covered by random timing | No action |
| All cores in C6 during DVFS | Fabric DVFS with no active core requestor | Covered by random stimulus | No action |
| Rapid C6 toggle at DVFS boundary | Entry/exit at exact DVFS transition point | Covered by random timing | No action |
| DVFS request queued during C6 | Request arrives while core clock gated | Covered by random stimulus | No action |
