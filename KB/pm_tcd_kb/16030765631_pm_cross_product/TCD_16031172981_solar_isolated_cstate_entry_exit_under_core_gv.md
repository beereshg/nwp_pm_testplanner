## Section 1: Architecture / Micro-architecture and Functionality

Validates that C-state entry/exit and core GV workpoint changes coexist correctly when no P-state stimulus is present. Isolates the C-state/GV interaction from P-state arbitration, testing the spec requirement that C-state changes have no impact on GV request completion.

> **Architecture overview:** See [TPF 22022562325](https://hsdes.intel.com/appstore/article-one/#/22022562325) Section 2

### NWP-Specific Deltas

- NWP 2 CBBs (96 cores); PkgC6 ZBB - limited to CC6
- **No IMH-side fabric DVFS** - Core GV is CBB-side only
- No P-state changes injected - isolates C-state/GV path

## Section 3: Reset, Power, and Clocking

- P-state held constant; C-state entry gates core clock domain
- GV operates on core domain; C-state exit triggers clock ungating
- Tests clock-domain boundary interaction in isolation

---

## Section 4: Programming Model

GV requests generated by PCode based on workload/power budget. C-state driven by OS idle. No P-state changes present - only GV as frequency variable. Tests spec requirement: C-State Change should not have impact on GV request.
