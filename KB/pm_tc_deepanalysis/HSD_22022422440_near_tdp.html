Section A: NWP Disposition & Justification

**Disposition: Runnable_On_N-1**

This test verifies the **NearTDP** feature using the PTU (Platform Test Utility) tool:

1. PTU iteratively increases voltage on Core, Fabrics, and other domains until power approaches TDP

2. Initiated via OS mailbox (uCR package-scoped register in TPMI space)

3. Primecode validates config, sends HPM commands via `CBB B2P_EXCHANGE`

4. Root primary propagates to secondary primecode (IMH die fabrics and adjacent CBBs)

On NWP (single iMH, 2 CBBs):

  • No secondary primecode for IMH (NWP has single iMH with single PCode instance)
  • `B2P_EXCHANGE` HPM path exists per CBB
  • TPMI uCR mailbox same mechanism

**Key Justification:**

  • NearTDP feature applicable on NWP
  • `plc.feature.p2` + `NGA_MAIN` + `PMSS_NWP_READINESS_CHECK` tags
  • PTU platform-agnostic tool
