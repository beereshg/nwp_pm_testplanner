1. PCT Feature Summary (NWP)

NWP Status: SUPPORTED ✅ — PCT (Priority Core Turbo) is one of only 2 SST features supported on NWP. All others (SST-PP/CP/BF) are ZBB'd.

Architecture

PCT uses SST-TF's CLOS-based frequency partitioning to elevate 8 HP cores to P0max (4.4 GHz target) while clipping the remaining 88 LP cores to ~P1. BIOS divides all 96 NWP cores (2 CBBs × 48) into 4 partitions of 24 cores; the first 2 cores per partition (lowest APIC ID in MADT order) become HP (CLOS[0]); all others become LP (CLOS[3]).

2. DLCP (Die Location Core Pinning) — Relevance Assessment

What is DLCP?

DLCP (Die Location Core Pinning) is a variant of PCT where HP core positions are fixed at specific die/module locations using the PCT_Module_Mask fuse, rather than being computed from APIC ID ordering at boot. This provides manufacturing-level determinism about which physical cores are always "HP".

NWP DLCP Status

⚠️ Confirmation needed: The NWP KB notes "NWP: DLCP not explicitly referenced — manufacturing fused core masks via CLOS used instead." The existing PCT TCs (22022422100–22022422118) do NOT test DLCP-specific behavior (SST_TF_INFO_10 readback, PCT_Module_Mask fuse verification). Two scenarios are possible:
  • Scenario A — DLCP present on NWP: PCT_Module_Mask fuse is programmed; SST_TF_INFO_10 is non-zero; HP core positions are manufacturing-fixed. → Current TCs miss DLCP-specific coverage.
  • Scenario B — DLCP absent on NWP: PCT_Module_Mask = 0; SST_TF_INFO_10 = 0; BIOS uses APIC-ID-order selection. → DLCP TCs would be N/A, but a negative check verifying SST_TF_INFO_10 = 0 is still useful.
Recommendation: DLCP is architecturally present in the NWP PCT HW path. A dedicated DLCP TCD is justified to either validate the feature (Scenario A) or confirm its absence with a negative check (Scenario B). Either way — the tests are needed.

TCD Architecture Section

DLCP (Die Location Core Pinning) is a PCT variant where HP core positions are fixed at manufacturing time via the PCT_Module_Mask fuse, rather than computed by BIOS at boot from APIC ID order. PrimeCode reads this fuse at Reset Phase 5 and populates SST_TF_INFO_10 TPMI register with the HP module mask. The OS/BIOS SST enumeration tool reads SST_TF_INFO_10 to discover which cores are manufacturing-designated HP cores.

Key functional difference vs standard PCT:

  • Standard PCT: BIOS picks HP cores from MADT order (flexible, changeable via BIOS knob)
  • DLCP PCT: HP cores are fuse-fixed; BIOS must honor the mask; SST_CLOS_ASSOC assignment follows PCT_Module_Mask
  • In DLCP mode: IA32_HWP_CAPABILITIES.highest_perf differs per core — HP reports P0max, LP reports LP clip ratio
