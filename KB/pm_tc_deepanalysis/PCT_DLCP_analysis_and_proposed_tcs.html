3 DLCP — What It Is

Standard PCT vs DLCP Variant

AspectStandard PCTDLCP Variant
HP core selectionBIOS-computed: first core per partition from MADT/APIC ID order; overridable via "PCT Core Select" BIOS knobFuse-fixed: PCT_Module_Mask OTP specifies exact die/module positions; BIOS must honor the mask
SST_TF_INFO_10Zero / unusedNon-zero: HP module mask exposed as RO TPMI register; populated by PrimeCode at Reset Phase 5
IA32_HWP_CAPABILITIESHP and LP cores may report same highest_perfHP reports P0max; LP reports LP_CLIP — per-core differentiation visible to OS
FlexibilityBIOS knob can shift HP offset within partitionNo software override; manufacturing decision
DiscoveryOS infers HP from topology/APICOS reads SST_TF_INFO_10 per die via SST TPMI enumeration

PrimeCode Role in DLCP

At Reset Phase 5, PrimeCode reads PCT_Module_Mask fuse and populates SST_TF_INFO_10 TPMI register per dielet via sst_tpmi_general.cpp::sstTfInfoInit(). BIOS then reads SST_TF_INFO_10 and maps HP cores to SST_CLOS_ASSOC accordingly.

Architecture Section (for TCD description)

DLCP is a PCT variant where HP core positions are fixed at manufacturing time via the PCT_Module_Mask OTP fuse. PrimeCode reads this fuse at Reset Phase 5 and populates the read-only SST_TF_INFO_10 TPMI register per CBB dielet via sst_tpmi_general.cpp::sstTfInfoInit(). BIOS then reads SST_TF_INFO_10 and must honor that mask when assigning SST_CLOS_ASSOC — HP cores per the mask → CLOS[0]; all others → CLOS[3]. In DLCP mode, IA32_HWP_CAPABILITIES.highest_perf differs per core (HP = P0max, LP = LP_CLIP), enabling OS software to distinguish HP from LP cores without parsing TPMI directly.

9 Final Recommendation

The current PCT validation suite is a good baseline and should be retained in full. However, it does not cover:
  1. DLCP presence vs absence (critical discriminator for NWP SKU)
  2. Fused HP-core discovery and validation via SST_TF_INFO_10
  3. Per-CBB-instance BIOS programming completeness (NWP-specific 2-CBB topology)
  4. MADT-based partition algorithm correctness on NWP's 96-core topology

A new PCT — DLCP TCD with 4 TCs closes these gaps. TC 1 acts as a Scenario A/B gate — it determines whether TCs 2–4 run as positive functional tests or negative confirmation tests. TC 3 and TC 4 are complementary NWP topology checks that cannot substitute for each other.
