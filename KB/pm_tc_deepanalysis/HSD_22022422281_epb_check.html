Section A: NWP Disposition & Justification

**Disposition: Runnable_On_N-1**

**EPB (Energy Performance Bias)** is a 4-bit value (0–15) that hints to PCode and system agents about the power/performance tradeoff:

  • 0–3: Performance
  • 4–7: Balanced Performance
  • 8–11: Balanced Power
  • 12–15: Power Saving

EPB source resolution (per `PCU_CR_POWER_CTL1[PWR_PERF_TUNING_CFG_MODE]`): BIOS control or OS control.

On NWP, the same EPB mechanism applies (`IA32_ENERGY_PERF_BIAS` MSR 0x1B0, `MISC_PWR_MGMT` control).

**Key Justification:**

  • EPB is an x86 architectural feature present on NWP
  • Same source resolution hierarchy (BIOS/OS control)
  • `DMR_PO` + `NGA_MAIN` tags: primary CI coverage
  • `PMSS_NWP_READINESS_CHECK` tag: evaluated for NWP

Section B: NWP-Specific Test Procedure

EPB Source Resolution


IF PCU_CR_POWER_CTL1[PWR_PERF_TUNING_CFG_MODE] == 1:
    EPB source = BIOS (BIOS controls IA32_ENERGY_PERF_BIAS)
ELSE:
    EPB source = OS (OS/driver writes IA32_ENERGY_PERF_BIAS per core)

Adapted Test Steps

Step Action NWP Adaptation
1 Run EPB verification `python runPmx.py -x nwp.xml -p epb -tM 60`
2 Set BIOS EPB mode (`PWR_PERF_TUNING_CFG_MODE = 1`) Read back EPB from BIOS-controlled register
3 Set EPB = 0 (Performance); verify agents use performance mode Memory controller, UPI, PCIe use performance EPB
4 Set EPB = 15 (Power Saving); verify agents use power saving Same agents switch to power-save config
5 Set OS EPB mode; issue per-core EPB via MSR `IA32_ENERGY_PERF_BIAS` (0x1B0) per core
6 Verify EPB reflected in relevant PCode decisions EET, HWP APS use EPB hint

Key Registers (NWP)


# NWP EPB verification

# EPB control mode
try:
    power_ctl1 = sv.socket0.imh0.punit.ptpcioregs.ptpcioregs.power_ctl1.read()
    cfg_mode = (power_ctl1 >> 12) & 1  # PWR_PERF_TUNING_CFG_MODE
    print(f"PWR_PERF_TUNING_CFG_MODE: {cfg_mode} (1=BIOS control, 0=OS control)")
except Exception as e:
    print(f"POWER_CTL1: {e}")

# Per-core EPB MSR (IA32_ENERGY_PERF_BIAS = 0x1B0)
for cbb_idx in range(2):
    cbb = getattr(sv.socket0, f'cbb{cbb_idx}')
    try:
        epb = cbb.compute0.cpu.module0.core0.ia32_energy_perf_bias.read()
        epb_val = epb & 0xF
        print(f"CBB{cbb_idx} core0 EPB: {epb_val} (0=Perf, 15=PowerSave)")
    except Exception as e:
        print(f"CBB{cbb_idx} EPB: {e}")

NWP Pass Criteria

  • EPB source correctly determined by `PWR_PERF_TUNING_CFG_MODE`
  • BIOS EPB mode: all cores use BIOS-programmed EPB value
  • OS EPB mode: per-core EPB values written via MSR are respected
  • System agents (memory controller, UPI on NWP) use EPB as hint
  • EET and HWP APS algorithm behavior shifts with EPB value
