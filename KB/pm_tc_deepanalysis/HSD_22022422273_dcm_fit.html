Section A: NWP Disposition & Justification

**Disposition: Runnable_On_N-1**

**DCM FIT** (Dual-Core Module Frequency Isolation Technology) provides frequency isolation between the two cores in a DCM that share the same PLL and FIVR. In DMR, each DCM has 2 cores sharing PLL/FIVR, so if one runs AVX-512 (high cdyn), both cores are forced to the AVX-512 frequency limit. DCM FIT mitigates this "noisy neighbor" effect.

**NWP Note**: NWP core micro-architecture must be verified for DCM topology. If NWP has the same PantherCove/BigCore design with 2-core DCMs, the mechanism is the same. The `PMSS_NWP_READINESS_CHECK` and `DMR_PO` tags confirm this is evaluated for NWP.

**Key Justification:**

  • `DMR_PO` tag: silicon validation bring-up priority
  • `NGA_MAIN` tag: primary CI coverage
  • `PMSS_NWP_READINESS_CHECK` tag: evaluated for NWP
  • DCM architecture must be confirmed for NWP before running
