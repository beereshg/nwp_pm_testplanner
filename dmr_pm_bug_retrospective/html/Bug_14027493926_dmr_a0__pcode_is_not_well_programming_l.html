Root Cause Summary

BIOS: OKSDCRB1.86B.0033.D20.2603110927

uCode: 0x80000997

When SST TF is enabled pCode is not programming correctly the LP ratios on WP4:

Following table shows the ratio programed in sv.socket0.cbb0.base.punit_regs.punit_gpsb.gpsb_infvnn_io_regs.wp_cv_ia_ccp_wp4_0[0]

We expect LP ratios discovered in SST_TF_INFO_0 to be programmed in wp_cv_ia_ccp_wp4_0

MSR 198 is resolving wrong ratios due this.

We see this as a regression issue since it is correctly set in:

BIOS: OKSDREL1.86B.0032.D77.2
