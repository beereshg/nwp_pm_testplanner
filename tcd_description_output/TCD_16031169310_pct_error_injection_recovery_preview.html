1. Architecture / Micro-architecture and Functionality

This TCD validates PCT error status reporting and recovery paths defined in the SST HAS: SST_CP_STATUS.ERROR_TYPE for unsupported/conflicting feature control, and EXCURSION_TO_MIN reporting/clear for CLOS groups hitting minimum frequency floor. > Architecture overview: See TPF 16030762939 for PCT architecture.

3. Reset, Power, and Clocking
  • ERROR_TYPE is set by PCode on control write rejection; persists until next successful control write.
  • EXCURSION_TO_MIN is set by PCode during ordered throttle when a CLOS group reaches its minimum floor.
  • Both are cleared by explicit SW action (new write or RESET_EXCURSION_TO_MIN).

7. Security / Safety / Policy
  • ERROR_TYPE is read-only from OS; cannot be spoofed.
  • EXCURSION_TO_MIN is read-only; only RESET_EXCURSION_TO_MIN is writable.
