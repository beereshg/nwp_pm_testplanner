1. Architecture / Micro-architecture and Functionality

PCT (Priority Core Turbo) is a distinct Intel PM feature that uses SST-TF CLOS-based frequency partitioning to designate a small subset of High Priority (HP) cores to operate at an elevated turbo frequency (up to P0max, ~4.4 GHz on NWP), while remaining Low Priority (LP) cores are clipped to approximately P1. PCT is tracked as its own feature (PCT HAS). Primary use case: CPU+GPU/accelerator systems where GPU-serving cores need maximum frequency without requiring other cores to sleep in C6. NWP scope: PCT is one of only 2 SST features active on NWP. SST-PP, SST-CP, SST-BF, HGS are ZBB'd (HSD 22021155414).

Key Design Points (PCT HAS §Solution)

  • **No C6 requirement**: HP cores achieve P0max with LP cores fully active at ~P1
  • **Default activation**: System boots with PCT enabled if supported; BIOS sets up CLOS before OS handoff
  • **HP selection algorithm**: Divide all available processors (in MADT order) into N partitions (default N=4); first consecutive cores in each partition = HP
  • **MADT ordering**: BIOS ensures cores in the same module are adjacent in MADT for natural module-scoped HP selection
  • **Runtime reconfiguration**: Users can use the Intel SST tool to enable/disable PCT or reassign HP/LP cores at OS runtime
  • **Virtualization**: VMM can assign HP logical host cores to specific VMs; all non-HP cores are limited to LP clip frequency

3. Reset, Power, and Clocking
  • PCT CLOS config (SST_CLOS_CONFIG/ASSOC/CP_CONTROL) must be reprogrammed after warm/cold reset
  • PrimeCode re-reads SST_TF fuses at reset Phase 5 and repopulates SST_TF_INFO_0/2/10
  • BIOS must re-program CLOS registers and MSR 0x1AD after each boot
  • PCT state does not survive warm reset — BIOS fully owns re-initialization

⚠ Reset cross-product coverage should be verified in individual TC descriptions.

4. Programming Model

Enabling Path (BIOS CPL3)

Boot-time programming sequence. Steps reference PCT HAS §BIOS and DMR Turbo HAS §PCT BIOS Requirements.

Boot sequence: PCT Enabling Flow
├─ PrimeCode (NIO reset Phase 5)
│   ├ reads SST_TF fuses + PCT_Module_Mask
│   └ writes SST_TF_INFO_0 (LP_CLIP), INFO_2 (HP TRL), INFO_10 (DLCP mask)
├─ BIOS CPL3 — if PCT Partition Count > 0
│   ├ divide all available processors (MADT order) into N partitions
│   ├ first consecutive cores per partition = HP; remainder = LP
│   ├ per CBB dielet (NWP: cbb0 + cbb1):
│   │   ├ SST_CLOS_CONFIG[0].min=P1, .max=INFO_2.RATIO_0  (HP)
│   │   ├ SST_CLOS_CONFIG[3].min=Pn, .max=INFO_0.LP_CLIP_RATIO_0 (LP)
│   │   ├ SST_CP_CONTROL.SST_CP_PRIORITY_TYPE = 1 (Ordered)
│   │   ├ SST_CLOS_ASSOC[core]: HP→CLOS[0], LP→CLOS[3]
│   │   └ SST_PP_CONTROL.feature_state[1] = 1 (SST-TF active)
│   └ MSR 0x1AD = INFO_2.RATIO_0 (HP TRL; 0xFF no longer valid, HSD 14025997048)
└─ Pcode runs standard SST-TF flow unchanged (PCT = SST-TF from Pcode perspective)

HP selection (NWP): 96 cores ÷ 4 partitions = 24 cores/partition
  Default: first 2 cores per partition = HP → 8 HP cores total

5. Operational Behavior

Test Cases

No TCs found.

Pass/Fail Criteria

  • PASS: SST_CP_CONTROL.sst_cp_enable = 1 after boot with PCT Partition Count > 0
  • PASS: HP core count = 8 on NWP (verified via SST_CLOS_ASSOC — 8 cores with CLOS[0])
  • PASS: BIOS menu exposes PCT HP Partition Count and PCT Core Selection knobs
  • PASS: SST_TF_INFO_10 non-zero if DLCP SKU; zero otherwise
  • FAIL: SST_CP_CONTROL.sst_cp_enable = 0 when PCT Partition Count > 0
  • FAIL: HP core count ≠ 8 (expected per CCB 14026595435)
  • FAIL: MSR 0x1AD value differs from SST_TF_INFO_2.RATIO_0
6. Corner Cases & Error Handling
  • PCT with Partition Count = 0 — graceful fallback to conventional turbo
  • PCT Partition Count not divisible evenly by cores — surplus PCT cores operate as non-PCT LP
  • DLCP SKU: PCT_Module_Mask = 0 — CLOS_ASSOC is honored (non-DLCP path)
  • Mutex: PCT + SST-BF must not both be enabled (DQ rule violation)
  • Mutex: PCT + FCT (Favored Core Turbo) must not both be enabled
  • Core disabled by BIST or BIOS — excluded from available processor count; affects HP partition math

⚠ Mutex and boundary scenarios should be detailed in individual TC descriptions.

7. Security / Safety / Policy
  • PCT is SoC-wide — cannot be selectively enabled per VM; if SST-TF is active, all non-HP cores are LP-clipped regardless of VM assignment
  • BIOS lock bit: TPMI SST lock bit prevents OS from overriding CLOS assignments post-CPL3
  • OS/VMM responsibility: affinitize high-priority workloads to HP cores; SW does not get automatic scheduling guarantees
  • DLCP: SW must discover HP core positions via IA32_HWP_CAPABILITIES or SST_TF_INFO_10; HP positions are fuse-fixed, not software-selectable
