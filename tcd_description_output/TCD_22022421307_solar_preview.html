1. Architecture / Micro-architecture and Functionality

Solar is Intel's silicon stress framework for C-state validation on NWP (PantherCove PNC). Solar rapidly exercises C-state entry/exit sequences across all cores, creating high-frequency C-state transitions to expose stability issues, firmware hang conditions, and residency counter errors. The Solar framework supports multiple test scopes (which cores are stressed) and modes (exercise-only or verify).

T2 Boundary Notes (2026-07-19)

  • This is a framework/stress TCD, not a primary owner of a single architectural WHAT.
  • Functional pass/fail bars stay owned by functional C-state TCDs.

2. Interfaces and Protocols

Solar runs as a user-space process on SVOS and controls C-state entry via:

  • Direct MWAIT/HALT instruction injection to target cores
  • PythonSV post-run register inspection
  • Solar log files for PASS/FAIL determination

Solar log output is in /path/to/solar.log — examine for FAIL, ERROR, TIMEOUT strings.

3. Reset, Power, and Clocking
  • Solar exercises high-frequency C-state transitions — this stresses the C-state entry/exit clock gating circuitry
  • Solar must be run with C-states enabled in BIOS; PkgC6 must be excluded from NWP Solar scope
4. Programming Model
# Post-Solar verification: check residency and ZBB compliance
import subprocess, sys
s = sv.socket0

# Run solar
ret = subprocess.run(["/usr/bin/solar/solar.sh",
    "/cstate", "-scope", "0:0:96", "-states", "C1,C6", "-r", "-mode", "e",
    "/logpath", "/tmp/solar_out", "/log_ip_disables"], timeout=300)

if ret.returncode != 0:
    print("FAIL: Solar returned non-zero")
    sys.exit(1)

# Verify PkgC6 ZBB
pkgc6 = s.uncore.msr.ia32_pkg_c6_residency.read()
assert pkgc6 == 0, f"FAIL: PkgC6 ZBB violated — residency={pkgc6:#x}"

# Verify C6 residency advanced on most cores
c6_active = sum(
    s.getbypath(f"cbb{c}.compute0.module0.core{k}.msr.ia32_c6_residency").read() > 0
    for c in range(2) for k in range(48)
)
print(f"PASS: Solar completed, C6 residency active on {c6_active}/96 cores, PkgC6=ZBB")

6. Corner Cases & Error Handling
  • Solar TIMEOUT: If Solar hangs (>60s), check for MCA or NLOG errors indicating firmware hang during C-state transition
  • Unsupported C-states: PkgC6 is ZBB on NWP — Solar must detect and gracefully handle attempts to enter PkgC6
  • High-frequency wake storm: Solar Exercise mode may trigger enough interrupts to prevent deep C-state entry — this is expected; verify with Verify mode separately
7. Security / Safety / Policy
  • Solar must not be run with PERF_GLOBAL_CTRL or debug counters enabled — they may interfere with C-state entry decisions
