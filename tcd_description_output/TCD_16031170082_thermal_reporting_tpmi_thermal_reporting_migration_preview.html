1. Architecture / Micro-architecture and Functionality

TPMI Thermal Reporting Migration validates that NWP thermal status and reporting registers have correctly migrated from legacy MSRs to TPMI MMIO equivalents, and that deprecated MSR paths behave as specified (reads return 0, writes ignored). > Architecture overview: See TPF — Thermal Status/Reporting Interfaces §Design Details.

Key Facts

  • Deprecated MSR reads return 0 (not #GP) — architectural guidance to avoid OS driver crashes
  • Deprecated MSR writes are ignored — no effect
  • NWP adds TPMI-only fields: DLCP PCT SST_TF_INFO registers, PMT telemetry aggregators (iMH PUNIT, D2D ULA & IO-CA)
  • PMT thermal counters: NWP-specific PMT objects for thermal telemetry
