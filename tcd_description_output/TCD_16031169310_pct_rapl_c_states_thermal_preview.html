1. Architecture / Micro-architecture and Functionality

This TCD validates PCT error status reporting and recovery paths defined in the SST HAS: SST_CP_STATUS.ERROR_TYPE for unsupported/conflicting feature control, and EXCURSION_TO_MIN reporting/clear for CLOS groups hitting minimum frequency floor. > Architecture overview: See TPF 16030762939 for PCT architecture.

3. Reset, Power, and Clocking
  • ERROR_TYPE is set by PCode on control write rejection; persists until next successful control write.
  • EXCURSION_TO_MIN is set by PCode during ordered throttle when a CLOS group reaches its minimum floor.
  • Both are cleared by explicit SW action (new write or RESET_EXCURSION_TO_MIN).

---

4. Programming Model

All error/recovery paths use the SST_CP_CONTROL → SST_CP_STATUS handshake. PCode processes the control write and reflects completion via LAST_HANDSHAKE.

ERROR_TYPE Encoding (SST_CP_STATUS bits 5:2)

CodeNameTriggerRegister State After
0x0No errorSuccessful SST-CP enable/disableERROR_TYPE = 0x0
0x1Not supportedEnable PCT on unfused/unsupported SKUERROR_TYPE = 0x1; feature remains disabled
0x2ConflictEnable PCT + SST-BF simultaneously (DQ violation)ERROR_TYPE = 0x2; original state preserved
0x3–0xFReserved——

EXCURSION_TO_MIN Flow (SST_CP_STATUS bits 11:8)

StepActorActionRegister Effect
1PCodeRAPL budget insufficient for CLOS group k → sets excursionEXCURSION_TO_MIN[k] = 1
2SWReads SST_CP_STATUS to detect which CLOS groups hit minRead bits 11:8
3SWWrites 0 to RESET_EXCURSION_TO_MIN[k] + sets HANDSHAKEControl write
4PCodeProcesses reset, clears excursion bit, reflects handshakeEXCURSION_TO_MIN[k] = 0, LAST_HANDSHAKE updated
5SWPolls LAST_HANDSHAKE to confirm completionReadback verify

7. Security / Safety / Policy
  • ERROR_TYPE is read-only from OS; cannot be spoofed.
  • EXCURSION_TO_MIN is read-only; only RESET_EXCURSION_TO_MIN is writable.
