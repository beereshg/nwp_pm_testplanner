1. Architecture / Micro-architecture and Functionality

CBB CCF Ring Freq Telemetry Counters validates the counter infrastructure that PCode uses to measure CCF ring frequency scalability inputs. Two counter groups are covered: (1) CBO lookup counters (ingress path, BW heuristic inputs) and (2) SBO snoop counters (snoop traffic per cluster, distress path inputs). The validation scope is counter accessibility -- specifically that disabled counters are frozen (value does not change over time) and that all counters can be bulk-disabled via the ingress_all register. Counter Disable Test Flow (both CBO and SBO):

  Step 1: Read initial counter value
          val_before = sv.cbbM.i_ccf_envs[k].ingresss[j].counter.read()

  Step 2: Disable the counter
          sv.cbbM.i_ccf_envs[k].ingresss[j].enable.write(0)

  Step 3: Wait rtime million cycles (default 100M = ~1 ms real time)

  Step 4: Read counter again
          val_after  = sv.cbbM.i_ccf_envs[k].ingresss[j].counter.read()

  Step 5: Pass condition: val_before == val_after   (counter is frozen)
          Fail condition: val_before != val_after   (counter incremented despite disabled)

  Step 6: Re-enable counter (cleanup)
          sv.cbbM.i_ccf_envs[k].ingresss[j].enable.write(1)


ingress_all Bulk Disable/Enable Test (CBO only):

  Repeat N=10 iterations:
    iter 1: ingress_all.write(0)  -> ALL ingresss[] disabled simultaneously
            wait -> verify ALL counters frozen
    iter 2: ingress_all.write(1)  -> ALL ingresss[] re-enabled
            wait -> verify ALL counters now accumulating

3. Reset, Power, and Clocking
  • Counter registers reset on warm reset; initial state = enabled
  • Counters only accumulate when the corresponding CBB is in active (C0) state
  • On emulation: use rtime in million cycles (typically 100M = ~1 ms at 100 MHz) as the wait interval

5. Operational Behavior

For each counter under test:

1. Read initial counter value

2. Disable the counter (set enable = 0)

3. Wait rtime million cycles

4. Read counter value again

5. Pass: values are identical (counter is frozen when disabled)

6. Re-enable counter

For ingress_all bulk test:

1. Iterate N times: disable all via ingress_all, verify frozen; re-enable all, verify accumulating

Counter wraps to 0 while enabledNot a failure; only disabled counters must be frozen
ingress_all partial writeAll counters in the group must be affected atomically
SBO counter in low-traffic scenarioUse sufficient rtime to ensure snoop traffic would normally cause increment

  • Counter read/write requires Ring 0 (PythonSV / PMX framework)
  • Counter disable is temporary; test restores enable state on exit
