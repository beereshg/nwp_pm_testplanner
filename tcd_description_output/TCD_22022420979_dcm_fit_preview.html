1. Architecture / Micro-architecture and Functionality

DCM FIT (Dual Core Module Frequency Isolation Tolerance) is an intra-die Active PM feature that prevents the "noisy neighbor" frequency coupling problem within a Dual-Core Module. In each DCM, two PantherCove (PNC) cores share the same PLL and FIVR. Without FIT, a heavy AVX-512 workload on Core 0 forces Core 1 (even if running a light SSE workload) to the same lower AVX-512 TRL floor, degrading QoS in multi-tenant/VM environments. DCM FIT allows software (BIOS or hypervisor) to program a per-DCM frequency variability tolerance via TPMI in units of 100 MHz relative to the SSE Turbo Frequency (SSE P0x). pCode PEGA enforces this tolerance so the lighter-workload core is allowed to run within FIT units of its SSE ceiling, regardless of the heavier core's ICCP license.

Key Design Points

  • FIT tolerance is programmed in units of 100 MHz below SSE P0x (SSE Turbo Frequency)
  • pCode evaluates per-DCM TPMI FIT tolerance each slow-loop evaluation (~1 mS)
  • When FIT = 0 (disabled): both cores coupled — Core 1 forced to AVX-512 TRL (default/baseline behavior)
  • When FIT = N: Core 1 frequency ≥ SSE P0x − N×100 MHz, independent of Core 0's ICCP license
  • FIT capability signaled via SST_PP_MISC_STATUS.MODULE_TURBO_CAPABILITY = 1 on supported SKUs
  • Feature is VMM-oriented: enables per-DCM frequency variability tolerance for hypervisors

3. Reset, Power, and Clocking
  • Boot: BIOS programs per-DCM FIT tolerance via TPMI during platform initialization; default is 0 (disabled)
  • Runtime: Hypervisor or BIOS may update TPMI FIT tolerance at runtime; pCode picks up new value on next slow-loop evaluation
  • No reset dependency: FIT is a software-configured policy; no hardware reset signal gating
  • Power domain: FIT operates within the existing per-module PLL/FIVR domain — no additional power plane

5. Operational Behavior

FIT Algorithm (pCode PEGA)

When FIT is active (tolerance > 0), pCode PEGA applies the following per-DCM rule each slow-loop:

  • Core 0 frequency ceiling: min(ICCP_TRL[AVX-512], Power/Thermal limits) — normal throttled path
  • Core 1 frequency ceiling: max(Core1_ICCP_TRL, SSE_P0x − FIT_tolerance × 100 MHz) — FIT floor prevents over-throttling

6. Corner Cases & Error Handling
  • FIT = 0 (disabled): both cores in DCM coupled at AVX-512 TRL — this is the baseline/regression scenario
  • Both cores AVX-512: FIT does not apply — both cores at same TRL by design
  • Core 1 idle: FIT not evaluated for idle cores; no isolation needed
  • FIT tolerance > (SSE P0x − AVX TRL): Core 1 effectively constrained to SSE P0x (best case); FIT ceiling may not be reachable
  • TPMI write after boot: pCode picks up new value next slow-loop; no reset required; verify TPMI readback

---

7. Security / Safety / Policy
  • FIT tolerance is configurable only by BIOS or hypervisor (privileged); OS userspace cannot modify TPMI
  • No security-sensitive registers; FIT is a QoS/performance isolation policy
  • Capability gated by MODULE_TURBO_CAPABILITY fuse bit; disabled SKUs cannot enable FIT
