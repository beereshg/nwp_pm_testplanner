TC: 16031105041  |  CBB CCF NonAutoGV Mode — Fast GV
Parent TCD: 22022421168 — CBB CCF PM GV Control Interface
Reformatted: 2026-07-15
Validation Scope

Verify that the CBB CCF ring operates correctly in NonAutoGV mode (the POR reset-default GV mode for CBB) using the Fast GV in FLL mode execution path. In NonAutoGV mode, CCF does not autonomously choose its target working point; instead it responds to CCF_WP register writes from PCode and executes the GV transition to the requested ratio.

This TC validates two execution sub-modes:

  1. Fast GV default — CCF PMA in FLL mode (CCF_UCLK_GV_CFG.legacy_gv_enable=0), FLL in its default frequency-crawl configuration. GV step: ratio change without UCLK stop. POR path for CBB.
  2. PLL crawling — CCF PMA in FLL mode but with PLL fuse override (pll_mode=1, pll_mode_ovrden=1), crawling via freq_crawl_delta_t / freq_crawl_delta_f. Survivability option.
⚠️ AutoGV is NOT POR for CBB. Drainless GV (legacy mode, UCLK stop) is also not POR since D2D does not support UCLK stop. Fast GV in FLL mode is the only POR path.

Pass / Fail Criteria
PASS

Fast GV default: all injected ratios reflected in UFS_STATUS.CURRENT_RATIO within timeout; GVFSM completes for all CBBs.
PLL crawling: ratio changes correctly via PLL crawl path with fuse override applied.
No GVFSM hang, no cross-CBB interference.

FAIL

Any ratio mismatch between injected CCF_WP and UFS_STATUS.CURRENT_RATIO; GVFSM stuck busy; PLL crawl not responding to fuse override.

Post-Process

Save: UFS_STATUS.CURRENT_RATIO per CBB at each GV step; GVFSM busy/done timing; fuse override register values for PLL crawling sub-test.
